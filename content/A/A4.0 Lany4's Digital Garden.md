# Lany4's Digital Garden
## Introduction
I built this website as a way to showcase my work across **multiple fields**, which would help me define who I am creatively while allowing people to see what kind of work I've been doing over the years. 

My experience with coding is very minimal. I can look at a CSS or HTML file and understand partly what is going on. As I'm writing this page on April 30, 2025, two days after I started this project, I still don't know what padding, flex, span, and display: block do. My background is in design, not in web development. 

Creating a website comes with a steep learning curve. The backend often contains lines upon lines of code that I could easily fall down the rabbit hole of trying to understand, but didn't have the time to. Building a more I wanted a more dynamic website with more interaction, optimal performance, and potentially API integration would likely take 3 to 6+ months of serious learning.

Cost was another constraint. I couldn't afford to hire a graphic designer or a web-developer. My only expense was $20 for the annual domain name, and everything else had to be built by me.

So what pushed me forward despite these limitations? A deadline. It all started with a school project: a professional poster due in April 2025. I wanted to include a QR code that would direct employers to my work. But then the question became: *what website should that QR code lead to?* That led to: *could I build one myself, with the help of ChatGPT?*

On the evening of April 24, 2025, I attempted to build a site using  Glitch, but ironically, it was glitchy and unstable. With the poster showcase event coming up in May, I pivoted quickly. I decided to build a site in Obsidian, a note-taking app I already understood, and publish it using Quartz. I liked the way it visually resembled the kinds of graph-based tools I've used before.

Throughout this process, ChatGPT helped me understand code, suggested best practices, and offered guidance on structure and tools. The words, design decisions, tools, and organization of this site are my own, but were strengthened through that collaboration.

The projects featured here reflect work I'm proud of, and the site will continue to evolve as I grow. This page, intentionally structured like a scientific paper, will walk through the tools I used, the reasoning behind my decisions, the features of the site, and, finally, a reflection on the outcome.
